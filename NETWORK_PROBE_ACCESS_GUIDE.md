# 网络探测页面访问指南

## 🚀 如何访问新的网络探测页面

### 1. 启动应用

首先确保前端和后端服务都在运行：

#### 启动后端服务
```bash
cd admin-mit-backend
python app.py
```

#### 启动前端服务
```bash
cd admin-mit-ui
npm run dev
```

### 2. 访问路径

新的网络探测系统可以通过以下方式访问：

#### 主要访问路径
- **主页面**: `http://localhost:3000/network`
- **直接访问**: 在浏览器地址栏输入上述URL

#### 通过导航菜单访问
1. 登录系统后，在左侧导航栏找到 **"网络运维"** 菜单
2. 点击展开后，可以看到以下子菜单：
   - 监控大屏
   - 探测管理  
   - 数据可视化
   - 配置构建器
   - 深度分析
3. 点击任意子菜单都会进入网络探测主页面
4. 在主页面内通过标签切换不同功能

### 3. 功能模块

进入网络探测页面后，你会看到5个主要标签：

#### 📊 监控大屏 (Dashboard)
- 实时状态监控
- 统计数据展示
- 自动刷新功能
- 全屏模式支持

#### 🔧 探测管理 (Probes)
- 探测任务列表
- 批量操作功能
- 快速测试
- 配置导入导出
- 高级筛选

#### 📈 数据可视化 (Visualizer)
- 协议分布图表
- 响应时间分析
- 状态分布统计
- 时间范围选择

#### 🛠️ 配置构建器 (Builder)
- 可视化配置创建
- 预设模板选择
- 实时配置验证
- 批量配置管理

#### 📊 深度分析 (Analytics)
- 性能趋势分析
- 异常检测
- SLA 监控
- 对比分析

### 4. 权限要求

访问网络探测功能需要以下权限：
- `network:read` - 查看网络探测（基础权限）
- `network:create` - 创建探测任务
- `network:update` - 编辑探测任务
- `network:delete` - 删除探测任务
- `network:execute` - 执行探测任务

### 5. 快速开始

1. **首次使用**：
   - 访问 `/network` 进入主页面
   - 点击 "配置构建器" 标签
   - 选择一个模板创建第一个探测任务

2. **查看监控**：
   - 创建探测任务后，切换到 "监控大屏" 标签
   - 查看实时状态和统计信息

3. **数据分析**：
   - 等待一段时间收集数据后
   - 切换到 "数据可视化" 或 "深度分析" 标签
   - 查看详细的性能分析报告

### 6. 故障排除

#### 如果无法访问网络探测页面：

1. **检查权限**：
   - 确保当前用户具有 `network:read` 权限
   - 联系管理员分配相应权限

2. **检查路由**：
   - 确保路由配置已更新
   - 重启前端开发服务器

3. **检查控制台错误**：
   - 打开浏览器开发者工具
   - 查看控制台是否有JavaScript错误
   - 查看网络请求是否正常

4. **清除缓存**：
   - 清除浏览器缓存
   - 硬刷新页面 (Ctrl+F5)

### 7. 开发调试

如果你是开发者，可以通过以下方式调试：

```bash
# 检查路由是否正确加载
console.log('Network routes loaded')

# 检查组件是否正确导入
import { NetworkProbePage } from './pages/Network'

# 检查权限是否正确配置
hasPermission('network:read')
```

### 8. 联系支持

如果遇到问题，请：
1. 检查上述故障排除步骤
2. 查看浏览器控制台错误信息
3. 查看后端服务日志
4. 联系技术支持团队

**注意**: 确保后端API服务正常运行，网络探测功能依赖后端提供的API接口。
